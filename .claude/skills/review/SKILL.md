# /review - Quick Review

## Step 1: Parallel Scan

Check all available sources in parallel:

1. **Todos**: Read today's `todos/YYYY-MM-DD/todos.md` for status
2. **Memory**: Check `memory/improvements.md` for pending items

If integration agents are available (check `.claude/agents/index.md`):
3. **GitHub**: PRs needing review, CI failures
4. **Issue Tracker**: New or updated issues
5. **Messages**: Unread mentions or DMs

## Step 2: Synthesize

Categorize findings:

### Needs Action Now
- Urgent items, blockers, failures

### Needs Action Today
- Open todos, pending reviews, follow-ups

### FYI
- Updates, completed items, informational

Keep the briefing concise. No fluff.
